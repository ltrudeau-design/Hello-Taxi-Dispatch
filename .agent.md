# Hello Taxi Dispatch — EauDev Context

## What This Is
A real-time taxi dispatch web application used in active production. Built with vanilla JS (7 kernels), Python WebSocket server (Qwen3.5-4B inference), and Google Maps API.

## ⚠️ Critical Rules
- **NEVER kill ports 8080 or 8765 if Hello Taxi is running** — this will disrupt live dispatch operations
- Always check `lsof -i:8080 -i:8765` before any server operations
- File edits to JS files are safe during a running session (browser has already loaded them)
- HTML file edits are safe — only take effect on next page load

## Starting Hello Taxi
Say "start Hello Taxi" or "HT up" in EauDev, or run:
```bash
cd "/Users/eaumac/Desktop/Code/Hello Taxi Dispatch" && bash START_HELLO_TAXI.command
```

Or double-click `START_HELLO_TAXI.command` in Finder.

## Stopping Hello Taxi
```bash
lsof -ti:8080,8765,8766 | xargs kill
```

## Architecture (All Kernels Audited & Fixed)
- **kernel-core.js (V108)** — App bootstrap, global DATA object, LLM abstraction, localStorage persistence
- **kernel-data.js (V109)** — Clients, place nicknames, roster, SMS generation
- **kernel-dispatch.js (V110)** — Dispatch form, LLM parsing, geocoding, queue management
- **kernel-live.js (V112)** — Live dispatch queue, driver cards, phase management
- **kernel-map.js (V111)** — Google Maps integration, routing, polylines, geocoding
- **kernel-timing.js (V113)** — ETA calculations, driver scoring, TimingService
- **kernel-preparser.js** — DEAD CODE, not loaded in V105+

## Servers
- **HTTP:** `python3 -m http.server 8080` — serves web app
- **WebSocket:** `python3 hello_taxi_ws_server.py` — Qwen3.5-4B inference on ws://localhost:8765
- **llama-server:** port 8766 — Qwen3.5-4B-Q4_K_M.gguf model
